## 13. APP ARCHITECTURE

### Files
- **`portugal-trip-2026.jsx`** — React JSX source (412 lines). The primary trip companion app
- **`portugal-trip-2026.html`** — Standalone HTML version for web hosting/sharing. Self-contained with React 18 + Babel from CDN
- **`Portugal-Remaining-ToDo.md`** — Running to-do list (maintained throughout planning sessions)

### Data Structure (JSX)
The app uses two main data arrays:

**`BDAYS`** — Barrocal on-estate activities tab. Array of day objects, each containing:
```
{ day: "Sat, Mar 28", label: "Day 1 — Arrival", sub: "optional subtitle", items: [
  { time, name, ppl: [A,N,R,C,J], price, cancel, note, flag }
]}
```

**`CAL`** — Main calendar tab. Array of day objects:
```
{ date, n: dayNumber, title, iso, hotel: "bar"|"cas"|"lis"|"fly", free: hoursOfFreeTime,
  note, prep, locs: [{w:[A,N,R,C,J], l:"bar"}], acts: [
  { t: time, n: name, p: [A,N,R,C,J], from: "A"|"B", src: 1, cost, loc, d: description,
    cancel, phone, mapQ, par: true, booked: true, links: [{t, u}] }
]}
```

**`ppl` array format:** `[Andy, Nicole, Rose, Christina, Jim]` — `1` = participating, `0` = not
**Colors:** `["#3B82F6","#EC4899","#F59E0B","#8B5CF6","#10B981"]`
**`from`:** `"A"` = Andy-arranged, `"B"` = Barrocal activity
**`src: 1`** = needs booking (shows "book" badge)
